GPG key

You can use my public GPG key for encrypting data you want to send to me privately and to check the authenticity of my signature in emails and tarball releases, for example. There are three main ways of obtaining my public GPG key:

  1. Using browser's inline GPG handling capabilities; Try it!
  2. Receiving keys from a GPG key server; Try it!
  3. Manually downloading a GPG key file; Try it!

After you obtain my public key, it's highly relevant that you verify the authenticity of the key by checking its fingerprint.

You might want to know that my GPG key is of type 2048-bit RSA, was generated on 2013-01-12 and won't expire. Its fingerprint is 7CB1 208C 7336 56B7 5962  2500 27B9 C6FD 28D6 18AF.

Receiving from a key server

My public GPG key can be found at hkp://keys.gnupg.net/ and its ID is 0x28D618AF. You can retrieve it by the following command:

{% highlight sh %} gpg --recv-keys 0x28D618AF --keyserver hkp://keys.gnupg.net/ {% endhighlight %}

Manual download

Download my armored public GPG key from this server and then import it to your public key-ring using:

{% highlight sh %} gpg --import oitofelix.gpg {% endhighlight %}

Checking fingerprint

To verify the authenticity of the key you've imported to your public key-ring, enter the command:

{% highlight sh %} gpg --fingerprint 0x28D618AF | sed -n '/^[[:blank:]]\+Key/s/^.*= //p' {% endhighlight %}

It must print the following fingerprint: 7CB1 208C 7336 56B7 5962  2500 27B9 C6FD 28D6 18AF

Otherwise something is wrong! In that case don't use the retrieved key to verify signatures or encrypt data, and contact me.

OpenSSH key

You can use my public OpenSSH key (2048-bit RSA) for granting me access to any computer you have an user account in and are allowed to make SSH connections to. To do that, follow the steps below logged in the system as the user you want to grant me access to.

First download my public OpenSSH key and verify the authenticity of the key by running:

{% highlight sh %} ssh-keygen -lf oitofelix.ssh | cut -f2 -d' ' {% endhighlight %}

It must print the following fingerprint: c6:11:b0:09:0d:b7:0e:4b:49:d1:36:88:da:33:a3:6e. Otherwise something is wrong! In that case abort the procedure at this step and contact me.

If the fingerprint is valid, you can safely add the key to your list of authorized OpenSSH keys by issuing the following command:

{% highlight sh %} cat oitofelix.ssh >> ~/.ssh/authorized_keys {% endhighlight %}

It's done! Contact me so we can proceed with the remote assistance.
